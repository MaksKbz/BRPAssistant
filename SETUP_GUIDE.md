## ШАГ 1. Установка Android Studio

### 1.1 Скачайте Android Studio

1. Откройте браузер и перейдите по ссылке:
   👉 **https://developer.android.com/studio**
2. Нажмите большую зелёную кнопку **"Download Android Studio"**
3. Примите условия лицензии (поставьте галочку) → нажмите **Download**
4. Файл весит ~1 ГБ — дождитесь окончания загрузки

### 1.2 Установите Android Studio

**Windows:**
1. Запустите скачанный файл `.exe`
2. Нажмите **Next** на всех экранах (настройки по умолчанию подходят)
3. Дождитесь окончания установки (может занять 10-20 минут)
4. Запустите Android Studio

**macOS:**
1. Откройте файл `.dmg`
2. Перетащите Android Studio в папку Applications
3. Запустите из Applications

**Linux (Ubuntu):**
```bash
# Распакуйте скачанный .tar.gz
tar -xvf android-studio-*.tar.gz
cd android-studio/bin
./studio.sh
```

### 1.3 Первоначальная настройка Android Studio

При первом запуске Android Studio спросит:
1. **Import Settings** → выберите **"Do not import settings"** → OK
2. **Data Sharing** → выберите **"Don't send"** (не отправлять статистику)
3. Дождитесь загрузки компонентов (SDK, Emulator и т.д.) — это может занять **10-30 минут**
4. Когда появится окно **"Welcome to Android Studio"** — всё готово!

## ШАГ 2. Установка Android SDK

SDK — это набор инструментов для разработки под Android.

### 2.1 Откройте SDK Manager

1. На стартовом экране Android Studio нажмите **More Actions** (три точки) → **SDK Manager**
   *(Или: File → Settings → Appearance & Behavior → System Settings → Android SDK)*

### 2.2 Установите нужную версию Android

1. Во вкладке **SDK Platforms**:
   - ✅ Поставьте галочку напротив **Android 15.0 (API 35)**
   - *(Если нет — поставьте галочку на **Android 14.0 (API 34)**)*
2. Нажмите **Apply** → дождитесь скачивания → **OK**

### 2.3 Установите SDK Tools

1. Перейдите во вкладку **SDK Tools**
2. ✅ Поставьте галочки:
   - **Android SDK Build-Tools 35** (или последняя версия)
   - **Android SDK Command-line Tools**
   - **Android Emulator**
   - **Android SDK Platform-Tools**
3. Нажмите **Apply** → дождитесь установки → **OK**

## ШАГ 3. Открытие проекта BRP Assistant

### 3.1 Скопируйте папку проекта

1. Найдите папку **BRPAssistant** (весь проект)
2. Скопируйте её в удобное место, например:
   - Windows: `C:\Projects\BRPAssistant`
   - macOS: `/Users/ВашеИмя/Projects/BRPAssistant`
   - Linux: `/home/ВашеИмя/Projects/BRPAssistant`

> ⚠️ **Важно:** В пути к проекту **не должно быть русских букв и пробелов!**
> ❌ Плохо: `C:\Мои Документы\BRPAssistant`
> ✅ Хорошо: `C:\Projects\BRPAssistant`

### 3.2 Откройте проект в Android Studio

1. Запустите Android Studio
2. На стартовом экране нажмите **"Open"** (или File → Open)
3. Выберите папку **BRPAssistant** (ту, где находится файл `build.gradle.kts`)
4. Нажмите **OK**

### 3.3 Дождитесь синхронизации Gradle

1. Android Studio начнёт **"Sync"** — скачивание библиотек
2. Внизу справа появится полоса прогресса
3. Первый раз это может занять **5-15 минут** (скачивается ~500 МБ библиотек)
4. Когда внизу появится надпись **"BUILD SUCCESSFUL"** или пропадёт прогресс-бар — всё готово!

> 💡 Если появилась ошибка — нажмите **File → Sync Project with Gradle Files** (кнопка 🐘 со стрелкой на панели)

## ШАГ 4. Исправление Gradle Wrapper

Проекту нужен Gradle Wrapper — скрипт для автоматической сборки. Android Studio создаст его автоматически, но если нужно вручную:

### 4.1 Через Android Studio (рекомендуется)

Android Studio сама предложит создать wrapper при первом открытии проекта. Просто нажмите **"Create"** или **"Fix"**, если появится предупреждение.

### 4.2 Вручную (если не создалось автоматически)

1. Внизу Android Studio нажмите вкладку **"Terminal"**
2. Введите команды:
   ```bash
   # Установите Gradle (если не установлен):
   # Windows: скачайте с https://gradle.org/install/
   # macOS: brew install gradle
   # Linux: sudo apt install gradle

   gradle wrapper --gradle-version 8.9
   ```
3. Нажмите Enter → дождитесь выполнения

## ШАГ 6. Подключение телефона для теста

### 6.1 Включите режим разработчика на телефоне

1. Откройте **Настройки** → **О телефоне**
2. Найдите пункт **"Номер сборки"** (или "Версия MIUI" на Xiaomi)
3. **Нажмите на него 7 раз подряд**
4. Появится уведомление "Вы стали разработчиком!"

### 6.2 Включите отладку по USB

1. Вернитесь в **Настройки**
2. Найдите новый раздел **"Для разработчиков"** (или "Параметры разработчика")
   - На некоторых телефонах: Настройки → Система → Для разработчиков
   - На Xiaomi: Настройки → Дополнительные настройки → Для разработчиков
3. Включите переключатель **"Отладка по USB"**
4. Подтвердите во всплывающем окне

### 6.3 Подключите телефон к компьютеру

1. Возьмите USB-кабель
2. Подключите телефон к компьютеру
3. На телефоне может появиться окно **"Разрешить отладку по USB?"**
   - ✅ Поставьте галочку **"Всегда разрешать с этого компьютера"**
   - Нажмите **"Разрешить"**
4. В Android Studio вверху появится название вашего телефона в выпадающем списке

> 💡 Если телефон не появляется:
> - Попробуйте другой USB-кабель (некоторые кабели — только для зарядки)
> - Установите драйверы производителя телефона на ПК
> - Попробуйте другой USB-порт

### Альтернатива: Эмулятор (без телефона)

Если телефона нет, можно использовать эмулятор:
1. В Android Studio: **Tools → Device Manager**
2. Нажмите **"Create Device"**
3. Выберите **Pixel 7** → Next
4. Выберите **API 34** (или 35) → Download → Next → Finish
5. Запустите эмулятор кнопкой ▶️

> ⚠️ Внимание: для работы LLM на эмуляторе нужно минимум 4 ГБ RAM, выделенные эмулятору!

## ШАГ 7. Сборка проекта

### 7.1 Первая сборка

1. В верхнем меню Android Studio нажмите:
   **Build → Make Project** (или комбинация **Ctrl+F9** / **Cmd+F9**)
2. Дождитесь окончания — внизу появится результат:
   - ✅ **"BUILD SUCCESSFUL"** — отлично, идём дальше!
   - ❌ **"BUILD FAILED"** — смотрите ошибки и исправляйте (см. Шаг 8)

### 7.2 Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| "Gradle sync failed" | Нажмите **File → Sync Project with Gradle Files** |
| "SDK not found" | Установите SDK (Шаг 2) |
| "Kotlin version mismatch" | Проверьте интернет, Gradle скачает нужную версию |
| "Cannot find task" | Нажмите **File → Invalidate Caches → Invalidate and Restart** |
| Красные подчёркивания в коде | Нажмите **File → Sync Project with Gradle Files** |

## ШАГ 9. Первый запуск приложения — что делать

После установки приложения на телефон:

### 9.1 Скачайте модель ИИ

Приложение работает **офлайн**, но сначала нужно скачать языковую модель:

1. На главном экране нажмите **«Модели ИИ»** (или значок шестерёнки)
2. Вы увидите список из 4 моделей:
   - 🟢 **Gemma 3 1B (Q4)** — ~530 МБ (самая маленькая, для слабых устройств)
   - ⭐ **Gemma 3 1B (Q8)** — ~1 ГБ (рекомендуется)
   - ⭐ **Gemma 3n E2B** — ~1.5 ГБ (лучшее качество)
   - 🔵 **Gemma 4 E2B** — ~1.5 ГБ (новейшая)
3. Нажмите **«Скачать»** рядом с выбранной моделью
4. Дождитесь окончания загрузки (нужен интернет!)
5. После скачивания нажмите **«Активировать»**

> 💡 Для первого теста рекомендуем **Gemma 3 1B (Q4)** — самая быстрая для скачивания.

### 9.2 Выберите вашу технику

1. Нажмите вкладку **«Техника»** (внизу)
2. Выберите **бренд** (Can-Am, Ski-Doo, Sea-Doo, Lynx)
3. Выберите **категорию** (ATV, SSV, Snowmobile, PWC...)
4. Выберите **модель** из каталога 2026 года
5. Подтвердите выбор

### 9.3 Начните диалог

1. Перейдите на вкладку **«Диагностика»** (внизу)
2. Опишите проблему, например: *"Двигатель не заводится на Outlander"* 
3. Или выберите один из быстрых симптомов
4. ИИ-ассистент ответит с пошаговой инструкцией

## 🚨 Частые проблемы и решения

### Проблема: "Cannot resolve symbol" / красный текст в коде
**Решение:**
1. Нажмите **File → Sync Project with Gradle Files**
2. Подождите завершения синхронизации
3. Если не помогло: **File → Invalidate Caches → Invalidate and Restart**

### Проблема: "Gradle sync failed: Connection timed out"
**Решение:**
1. Проверьте интернет-соединение
2. Если вы в России/Казахстане — может понадобиться VPN
3. Попробуйте: **File → Settings → Gradle → снимите "Offline work"**

### Проблема: "INSTALL_FAILED_UPDATE_INCOMPATIBLE" на телефоне
**Решение:**
1. Удалите приложение BRP Assistant с телефона вручную
2. Нажмите Run снова

### Проблема: Приложение запускается, но чат не отвечает
**Решение:**
1. Перейдите в раздел «Модели ИИ»
2. Скачайте и активируйте модель
3. Без модели ИИ чат работать не будет

### Проблема: Модель не скачивается
**Решение:**
1. Проверьте интернет на телефоне
2. Модели скачиваются с HuggingFace — может быть медленно
3. Попробуйте модель поменьше (Gemma 3 1B Q4 — 530 МБ)
